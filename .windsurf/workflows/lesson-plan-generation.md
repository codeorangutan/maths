# Comprehensive Lesson Plan Generation Workflow

## Purpose
Generate full teaching documents for each lesson following the Lesson 12 format with complete pedagogical structure.

## Lesson Structure Template

Each lesson file must include:

### 1. Program Stage
- Year: 10/11/12 (accelerated stream)
- Term: 1-4
- Week: 1-10
- Session: Lesson XX
- Duration: 100 minutes

### 2. Lesson Title
Descriptive title capturing the core focus

### 3. Lesson Purpose
One paragraph explaining why this lesson exists and what it achieves

### 4. Learning Intentions
4-5 specific outcomes starting with "By the end of this lesson, the student will:"

### 5. Success Criteria
Measurable benchmarks with percentages/timing targets

### 6. Minute-by-Minute Flow (100 min)
Structured sections with:
- Setup/Reset (8 min)
- Core Content Block (40 min) - with 10 specific problems
- Challenge Block (15 min) - with 3-5 harder problems
- Timed Sprint (15 min) - ATAR-style exam practice
- Error Audit/Repair (14 min)
- Exit/Homework Setup (8 min)

### 7. Core Content Block (10 problems)
Problems must:
- Cover lesson's key concepts
- Include definitions, formulas, worked methods
- Show step-by-step solutions
- Use proper MathJax formatting

### 8. Challenge Block (3-5 problems)
- AOPS/Alcumus style extension problems
- Non-repeating across lessons
- Lesson-topic aligned

### 9. Timed Sprint (3 problems)
- Exam-style pressure practice
- Mixed difficulty

### 10. Solutions
Complete worked solutions for ALL problems with:
- Clear steps
- Formula applications
- Final answers boxed/highlighted

### 11. Homework
- Core: 3-4 consolidation tasks
- Extension: 2-3 challenge tasks

### 12. Teacher Diagnostic Notes
- Tracking metrics
- Intervention rules
- Next lesson links

## Content Requirements

### Definitions
Every key term must be:
- Precisely defined
- Mathematically rigorous
- Student-friendly language

### Formulas
All formulas must:
- Be stated clearly before use
- Include domain restrictions
- Show derivation where relevant
- Use proper LaTeX formatting

### Methods
Each problem type needs:
- General solution strategy
- Step-by-step algorithm
- Common pitfalls warning
- Verification checks

### Examples
Worked examples must:
- Show complete working
- Include reasoning at each step
- Highlight key decision points
- Present multiple approaches when useful

## Generation Process

For each lesson:
1. Analyze the topic from syllabus map
2. Identify prerequisite knowledge
3. Define 4-5 core concepts
4. Create 10 progressive core problems
5. Design 3-5 challenge extensions
6. Build 3 timed sprint problems
7. Write complete solutions
8. Add diagnostic tracking
9. Validate MathJax formatting
10. Check against prior lessons (no repeats)

## Quality Checks

- [ ] All math properly formatted with MathJax
- [ ] 10 core problems present with solutions
- [ ] 3+ challenge problems present with solutions
- [ ] 3 timed sprint problems present with solutions
- [ ] Homework section complete
- [ ] Teacher diagnostics included
- [ ] No question repeats from previous lessons
- [ ] Error categories (C,P,A,R,T) referenced
- [ ] Strand codes (NA,FG,TG,PS,RP,TX) referenced
